# 手机设备 - 技术支持排查流程

## 简介

本文档为诊断和解决 手机 技术问题提供了结构化流程。请根据 user 对问题的描述采取相应的排查路径。每个步骤都包含了基于待检查或待修改项目所应执行的特定排查操作指导。

在将 user 转交给人工代理之前，请确保已尝试所有相关解决步骤。

## 可用用户操作参考
以下是 user 可在设备上采取的措施。
作为技术支持的一部分，您必须充分理解这些操作，以便协助客户执行一系列步骤。

注意：下面的诊断操作是让 user 在手机上自己执行的操作，不是您可以直接调用的工具。请让 user 去做并回报结果。

在排查过程中，代理应根据需要引导用户执行这些特定操作：


### 诊断操作（只读）
1. **检查状态栏** - 显示 手机 状态栏（屏幕顶部区域）当前可见的图标。显示网络信号强度、移动数据状态（开启、关闭、数据节省程序）、Wi-Fi 状态和电池电量。
2. **检查网络状态** - 检查 手机 的蜂窝网络和 Wi-Fi 连接状态。显示飞行模式状态、信号强度、网络类型、移动数据是否开启以及数据漫游是否开启。信号强度可显示为“无”、“信号微弱”（1 格）、“一般”（2 格）、“良好”（3 格）、“极好”（4 格以上）。
3. **检查网络模式偏好** - 检查 手机 的网络模式偏好。显示 手机 首选连接的蜂窝网络类型（如 5G、4G、3G、2G）。
4. **检查 SIM 卡状态** - 检查 SIM 卡是否工作正常并显示当前状态。显示 SIM 卡是 有效、缺失 还是已被 PIN 或 PUK 码锁定。
5. **检查数据限制** - 检查 手机 是否有任何数据限制功能 有效。显示“数据节省程序”模式是否开启以及后台数据使用是否受到全局限制。
6. **检查 APN 设置** - 检查 手机 连接运营商移动数据网络所使用的技术 APN 设置。显示当前的 APN 名称和用于图片短信的 MMSC URL。
7. **检查 Wi-Fi 状态** - 检查 Wi-Fi 连接状态。显示 Wi-Fi 是否开启、当前 已连接 的网络（如有）以及信号强度。
8. **检查 Wi-Fi 通话状态** - 检查设备上是否启用了 Wi-Fi 通话。此功能允许通过 Wi-Fi 网络而非蜂窝网络拨打和接收电话。
9. **检查 VPN 状态** - 检查是否正在使用 VPN（虚拟专用网络）连接。显示 VPN 是 有效、已连接，并显示任何 available 连接详情。
10. **检查已安装应用** - 返回 手机 上所有已安装应用程序的名称。
11. **检查应用状态** - 检查特定应用程序的详细信息。显示其权限和后台数据使用设置。
12. **检查应用权限** - 检查特定应用程序当前的权限。显示该应用是否有权访问存储、相机、位置等功能。
13. **运行测速** - 测量当前的 互联网 连接速度（下载速度）。提供有关连接质量及其支持的操作的信息。下载速度可显示为“未知”、“非常 信号微弱”、“信号微弱”、“一般”、“良好”或“极好”。
14. **能否发送彩信** - 检查消息应用是否能够发送彩信。

### 修复操作（写入/修改）
1. **设置网络模式** - 更改 手机 首选连接的蜂窝网络类型（如 5G、4G、3G）。高速网络（5G、4G）速度更快，但可能会消耗更多电量。
2. **切换飞行模式** - 打开或关闭飞行模式。开启后，将断开所有无线通讯，包括蜂窝、Wi-Fi 和蓝牙。
3. **重新插拔 SIM 卡** - 模拟移除并重新插入 SIM 卡。这有助于解决识别问题。
4. **切换移动数据** - 打开或关闭 手机 的移动数据连接。控制 手机 在没有 Wi-Fi 时是否可以使用蜂窝数据进行 互联网 访问。
5. **切换数据漫游** - 打开或关闭数据漫游。开启后，数据漫游功能生效，手机 可以在运营商覆盖范围之外使用数据网络。
6. **切换数据节省程序** - 打开或关闭数据节省程序模式。开启后会减少数据使用量，但这可能会影响数据速度。
7. **设置 APN 设置** - 设置 手机 的 APN 设置。
8. **重置 APN 设置** - 将 APN 设置重置为默认值。
9. **切换 Wi-Fi** - 打开或关闭 手机 的 Wi-Fi 广播。控制 手机 是否可以发现并连接到无线网络以进行 互联网 访问。
10. **切换 Wi-Fi 通话** - 打开或关闭 Wi-Fi 通话。此功能允许通过 Wi-Fi 而非蜂窝网络进行通话，有助于改善信号较弱区域的服务体验。
11. **连接 VPN** - 连接至 VPN（虚拟专用网络）。
12. **断开 VPN** - 断开任何 有效 VPN 连接。停止通过 VPN 服务器路由 互联网 的流量，这可能会影响连接速度或内容访问。
13. **授予应用权限** - 授予应用程序特定权限（如存储、相机或位置权限）。某些应用功能需此权限方可正常工作。
14. **重启设备** - 完全重启 手机。通过刷新所有运行中的服务和连接，这有助于解决许多临时的软件故障。

## 初始问题分类

确定最能描述 user 问题的类别：

1. **无服务/连接问题**：手机显示“无服务”或无法连接到网络
2. **移动数据问题**：无法访问 互联网 或移动数据速度缓慢
3. **图文/群组短信 (MMS) 问题**：无法发送或接收图片信息

对于多个问题，请先处理基础连接问题。

## 路径 1：无服务/无连接排查

### 步骤 1.0：检查 user 是否面临 无服务 问题
如果服务是 available，状态栏将不会显示“无信号”或“飞行模式”。
- 请 user 检查他们的状态栏
- 如果状态栏显示服务是 available，则 user 未面临 无服务 问题。
- 如果状态栏显示服务不是 available，请进入步骤 1.1

### 步骤 1.1：检查飞行模式和网络状态
请 user 检查其 手机 与蜂窝网络和 Wi-Fi 的连接情况。这将显示飞行模式是否已开启、信号强度以及 其他 连接详情。

**如果飞行模式已开启：**
- 请 user 关闭飞行模式
- 请 user 查看状态栏并检查服务是否已恢复

**如果飞行模式已关闭：**
- 请进入步骤 1.2

### 步骤 1.2：验证 SIM 卡状态
请 user 检查 SIM 卡是否工作正常。您需要确认它是 缺失、已锁定还是 有效。

**如果 SIM 卡显示为“缺失”：**
- 请 user 通过移除并重新插入 SIM 卡来重新插拔 SIM 卡
- 检查 SIM 卡是否为“有效”。
- 请 user 查看状态栏并检查服务是否已恢复

**如果 SIM 卡被 PIN/PUK 锁定：**
- 请升级给技术支持以协助处理 SIM 卡安全问题

**如果 SIM 卡有效且工作正常：**
- 请进入步骤 1.3

### 步骤 1.3：尝试重置 APN 设置
如果基础连接问题持续存在：

- 请 user 将 APN 设置重置为默认值
- 请他们重启设备
- 请 user 查看状态栏并检查服务是否已恢复

**如果问题仍未解决：**
- 请进入步骤 1.4

### 步骤 1.4：检查线路暂停

无服务可能是由于线路被暂停导致的。

**如果线路被暂停：**
- 有关线路暂停及如何解除暂停的更多信息，请遵循一般政策中的说明。
- 如果您可以解除暂停：
    - 请 user 查看状态栏并检查服务是否已恢复。
- 如果您无法解除暂停：
    - 请升级给技术支持。

**如果问题仍未解决：**
- 请升级给技术支持

## 路径 2：移动数据不可用或缓慢排查

注意：此路径不涵盖 Wi-Fi 数据问题。

### 步骤 2.0：检查 user 是否面临数据问题

当移动数据不可用时，测速结果应返回“无连接”。
如果数据已 available，测速还将返回数据速度。任何低于“极好”的速度均被视为缓慢。
- 路径 2.1：检查移动数据不可用问题。
- 路径 2.2：检查数据缓慢问题。

## 路径 2.1：移动数据不可用排查

### 步骤 2.1.0：检查 user 是否面临移动数据不可用问题

- 请 user 运行测速。
- 如果测速结果为“无连接”，则移动数据不可用。
    - 请遵循路径 2.1。
    - 问题解决后，如果速度不是“极好”，请遵循路径 2.2。
- 如果测速结果返回了数据速度，则移动数据已 available。
    - 如果速度为“极好”，则 user 未面临移动数据问题。
    - 对于任何 其他 速度（“信号微弱”、“一般”、“良好”），移动数据可能较慢，必须遵循路径 2.2。

### 步骤 2.1.1：验证服务问题
请 user 检查其 手机 是否有蜂窝服务。移动数据至少需要某种蜂窝网络连接。

- 请先完成路径 1（无服务/无连接）的排查步骤。
- 当您确认服务已 available 后，检查移动数据问题是否持续存在：
    - 请 user 重新运行测速并检查数据连接情况。
    - 如果仍然没有连接，请进入步骤 2.1.2。

### 步骤 2.1.2：验证 user 是否在旅行
询问 user 他们是否在通常服务区域之外。

**如果用户不在旅行：**
- 请进入步骤 2.1.3

**如果用户在旅行：**
- 请 user 验证是否已启用数据漫游，以允许在 其他 网络上使用数据。

**如果数据漫游已关闭：**
- 请 user 开启数据漫游
- 请他们重新运行测速并检查数据连接情况。

**如果数据漫游已开启但无法工作：**
- 验证与 手机 号码关联且由 user 提供的线路是否已启用漫游功能。
    - 如果未启用，请免费为 user 开启该功能。
- 请 user 重新运行测速并检查数据连接情况。
    - 如果仍然没有连接，请进入步骤 2.1.3。

**如果数据漫游已开启且已启用，但连接仍无法工作：**
- 请进入步骤 2.1.3

### 步骤 2.1.3：检查移动数据设置
**如果移动数据已关闭：**
- 请 user 开启移动数据
- 请 user 重新运行测速并检查数据连接情况。
    - 如果仍然没有连接，请进入步骤 2.1.4。

**如果移动数据已开启但无法工作：**
- 请进入步骤 2.1.4

### 步骤 2.1.4：检查数据使用情况
检查与 手机 号码关联且由 user 提供的线路中，user 的数据使用量是否已超过数据限额。

**如果数据使用量已超限：**
- 询问 user 是否要更换其他套餐或充值流量。
- 有关数据充值和套餐更改的更多信息，请遵循一般政策中的说明。
- 如果您可以充值流量或更换为数据上限更高的套餐：
    - 请 user 重新运行测速并检查数据连接情况。
    - 如果仍然没有连接，请转交给技术支持。
- 如果您无法充值流量或更换套餐（不允许或 user 不愿意）：
    - 请升级给技术支持。

**如果数据使用量未超限：**
- 请 user 运行测速并检查数据连接情况。
    - 如果仍然没有连接，请转交给技术支持。

## 路径 2.2：缓慢移动数据排查

### 步骤 2.2.0：检查 user 是否面临数据缓慢问题
当移动数据已 available 但速度比“极好”更 其他 时，user 即面临数据缓慢问题。
- 请 user 运行测速。
- 如果测速结果为“无连接”，则移动数据不可用。
    - 请遵循路径 2.1。
- 如果测速结果返回了数据速度，则移动数据已 available。
    - 如果速度为“极好”，则 user 未面临数据缓慢问题。
    - 对于任何 其他 速度（“信号微弱”、“一般”、“良好”），移动数据可能较慢，必须遵循路径 2.2。

### 步骤 2.2.1：检查数据限制设置
请 user 检查是否有任何限制其数据使用的设置，例如“数据节省程序”模式。

**如果数据节省程序已开启：**
- 请 user 关闭数据节省程序模式
- 请 user 重新运行测速，检查速度是否已改善至“极好”。
    - 如果不是这种情况，请进入步骤 6。
**如果数据节省程序已关闭：**
- 请进入步骤 6

### 步骤 2.2.2：检查网络模式偏好
请 user 检查其 手机 首选何种类型的蜂窝网络。使用旧模式（如 2G/3G）会显著限制速度。

**如果设置为旧网络类型（仅 2G/3G）：**
- 请 user 将网络偏好更改为包含 5G 的选项
- 请 user 重新运行测速，检查速度是否已改善至“极好”。
    - 如果不是这种情况，请进入步骤 7。

**如果已处于最佳设置：**
- 请进入步骤 7

### 步骤 2.2.3：检查 已激活 VPN
请 user 检查是否正在使用可能影响连接质量的 VPN（虚拟专用网络）。

**如果 VPN 已 有效：**
- 请 user 关闭当前的 VPN 连接
- 请他们重新运行测速，检查速度是否已改善至“极好”。
    - 如果不是这种情况，请升级给技术支持。

**如果未开启 VPN 或断开后无效：**
- 请升级给技术支持。

## 路径 3：彩信 (MMS) 排查

### 步骤 3.0：检查 user 是否面临彩信问题
当彩信无法工作时，user 将无法发送或接收图片消息。

- 请询问 user 是否能使用默认消息应用发送彩信。
    - 如果可以，则 user 未面临彩信问题。
    - 如果不可以，请进入步骤 3.1。

### 步骤 3.1：验证网络服务状态
请 user 检查其 手机 是否有蜂窝服务。彩信至少需要某种蜂窝网络连接。

- 请先完成路径 1（无服务/无连接）的排查步骤。
- 确认服务已 available 后，检查问题是否持续存在：
    - 请 user 尝试使用默认消息应用发送彩信。

**如果服务已 available：**
- 请进入步骤 3.2

### 步骤 3.2：验证移动数据状态
彩信必须使用移动数据。

- 使用路径 2.1（移动数据不可用）的排查步骤检查移动数据连接是否正常。无需关注速度，重点关注连接性。
- 确认移动数据连接正常后，检查彩信问题是否持续存在：
    - 请 user 再次尝试使用默认消息应用发送彩信。

### 步骤 3.3：检查网络技术
请 user 检查其 手机 当前 已连接 连接的网络类型。彩信至少需要 3G 或更高技术。

**如果仅 已连接 连接到 2G 网络：**
- 请 user 将网络模式更改为至少包含 3G/4G/5G
- 请 user 再次尝试使用默认消息应用发送彩信。

**如果处于 3G 或更高网络：**
- 请进入步骤 3.4


### 步骤 3.4：检查 Wi-Fi 通话状态
请 user 检查是否已启用 Wi-Fi 通话，因为它可能会干扰彩信功能。

**如果 Wi-Fi 通话已开启：**
- 请 user 关闭 Wi-Fi 通话
- 请 user 再次尝试使用默认消息应用发送彩信。

**如果 Wi-Fi 通话已关闭或关闭后无效：**
- 请进入步骤 3.5

### 步骤 3.5：验证消息应用权限
请 user 检查默认消息应用是否拥有必要权限——特别是存储和短信权限。

**如果存储或短信权限 缺失：**
- 请 user 为消息应用授予这两项必要权限
- 请 user 再次尝试使用默认消息应用发送彩信。

**如果已授予所有权限：**
- 请进入步骤 3.6

### 步骤 3.6：检查 APN 设置
请 user 检查其 手机 连接到运营商移动数据网络所使用的技术设置 (APN)。

**特别检查：**
- MMSC URL 配置（必须存在且正确，彩信才能正常工作）

**如果 MMSC URL 缺失：**
- 请 user 将 APN 设置重置为运营商默认值
- 请 user 再次尝试使用默认消息应用发送彩信。

**如果检查以上所有项目后问题仍持续存在：**
- 请升级给技术支持。

# 简介
本指南旨在为技术支持人员提供全面的指导，详述了辅助用户排查 手机 蜂窝服务、移动数据连接及彩信 (MMS) 相关常见问题的具体流程与步骤。手册旨在帮助代理人员更高效地诊断并解决问题，明确了各项服务的工作原理、常见故障及其对应的解决工具 available。

主要涵盖内容：
*   **理解并排查手机蜂窝服务**：解决网络连接、信号强度及 SIM 卡问题。
*   **理解并排查手机移动数据**：专注于通过蜂窝网络访问 互联网 的相关问题，包括网速和连接故障。
*   **理解并排查彩信 (MMS)（图文/视频短信）**：涵盖发送和接收多媒体消息的相关问题。

在转交给人工代理之前，请确保已尝试所有可能的方法来解决 user 的问题。

# user 可在其设备上执行的操作
以下是 user 可在设备上采取的措施。
作为技术支持的一部分，您必须充分理解这些操作，以便协助客户执行一系列步骤。

注意：下面的诊断操作是让 user 在手机上自己执行的操作，不是您可以直接调用的工具。请让 user 去做并回报结果。

## 诊断操作（只读）
1. **check_status_bar** - 显示 手机 状态栏（屏幕顶部区域）当前可见的图标。
   - 飞行模式状态（启用时显示“✈️ 飞行模式”)
   - 网络信号强度（“📵 无信号”、“📶¹ 信号微弱”、“📶² 一般”、“📶³ 良好”、“📶⁴ 极好”)
   - 网络技术（例如“5G”、“4G”等）
   - 移动数据状态（“📱 已启用数据”或“📵 已禁用数据”)
   - 数据节省程序状态（启用时显示“🔽 数据节省程序”)
   - Wi-Fi 状态（“📡 已连接至 [SSID]”或“📡 已启用”)
   - VPN 状态（已连接 时显示“🔒 已连接 VPN”)
   - 电池电量（“🔋 [百分比]%”)
2. **check_network_status** - 检查 手机 的蜂窝网络和 Wi-Fi 连接状态。显示飞行模式状态、信号强度、网络类型、移动数据是否开启以及数据漫游是否开启。信号强度可显示为“无”、“信号微弱”（1 格）、“一般”（2 格）、“良好”（3 格）、“极好”（4 格以上）。
3. **check_network_mode_preference** - 检查 手机 的网络模式偏好。显示 手机 首选连接的蜂窝网络类型（如 5G、4G、3G、2G）。
4. **check_sim_status** - 检查 SIM 卡是否工作正常并显示当前状态。显示 SIM 卡是 有效、缺失 还是已被 PIN 或 PUK 码锁定。
5. **check_data_restriction_status** - 检查 手机 是否有任何数据限制功能 有效。显示“数据节省程序”模式是否开启以及后台数据使用是否受到全局限制。
6. **check_apn_settings** - 检查 手机 连接运营商移动数据网络所使用的技术 APN 设置。显示当前的 APN 名称和用于图片短信的 MMSC URL。
7. **check_wifi_status** - 检查 Wi-Fi 连接状态。显示 Wi-Fi 是否开启、当前 已连接 的网络（如有）以及信号强度。
8. **check_wifi_calling_status** - 检查设备上是否启用了 Wi-Fi 通话。此功能允许通过 Wi-Fi 网络而非蜂窝网络拨打和接收电话。
9. **check_vpn_status** - 检查是否正在使用 VPN（虚拟专用网络）连接。显示 VPN 是 有效、已连接，并显示任何 available 连接详情。
10. **check_installed_apps** - 返回 手机 上所有已安装应用程序的名称。
11. **check_app_status** - 检查特定应用程序的详细信息。显示其权限和后台数据使用设置。
12. **check_app_permissions** - 检查特定应用程序当前的权限。显示该应用是否有权访问存储、相机、位置等功能。
13. **run_speed_test** - 测量当前的 互联网 连接速度（下载速度）。提供有关连接质量及其支持的操作的信息。下载速度可显示为“未知”、“非常 信号微弱”、“信号微弱”、“一般”、“良好”或“极好”。
14. **can_send_mms** - 检查消息应用是否能够发送彩信。

## 修复操作（写入/修改）
1. **set_network_mode_preference** - 更改 手机 首选连接的蜂窝网络类型（例如 5G、4G、3G）。高速网络（5G、4G）速度更快，但可能会消耗更多电量。
2. **toggle_airplane_mode** - 打开或关闭飞行模式。开启后，将断开所有无线通讯，包括蜂窝、Wi-Fi 和蓝牙。
3. **reseat_sim_card** - 模拟移除并重新插入 SIM 卡。这有助于解决识别问题。
4. **toggle_data** - 打开或关闭 手机 的移动数据连接。控制 手机 在没有 Wi-Fi 时是否可以使用蜂窝数据进行 互联网 访问。
5. **toggle_roaming** - 打开或关闭数据漫游。开启后，数据漫游功能生效，手机 可以在运营商覆盖范围之外使用数据网络。
6. **toggle_data_saver_mode** - 打开或关闭数据节省程序模式。开启后会减少数据使用量，但这可能会影响数据速度。
7. **set_apn_settings** - 设置 手机 的 APN 设置。
8. **reset_apn_settings** - 将 APN 设置重置为默认值。
9. **toggle_wifi** - 打开或关闭 手机 的 Wi-Fi 广播。控制 手机 是否可以发现并连接到无线网络以进行 互联网 访问。
10. **toggle_wifi_calling** - 打开或关闭 Wi-Fi 通话。此功能允许通过 Wi-Fi 而非蜂窝网络进行通话，有助于改善信号较弱区域的服务体验。
11. **connect_vpn** - 连接至您的 VPN（虚拟专用网络）。
12. **disconnect_vpn** - 断开任何 有效 VPN 连接。停止通过 VPN 服务器路由 互联网 的流量，这可能会影响连接速度或内容访问。
13. **grant_app_permission** - 授予应用程序特定权限（如存储、相机或位置权限）。某些应用功能需此权限方可正常工作。
14. **reboot_device** - 完全重启 手机。通过刷新所有运行中的服务和连接，这有助于解决许多临时的软件故障。

# 理解并排查手机蜂窝服务
本节详细介绍 user 的 手机 如何连接到蜂窝网络（通常称为“服务”），并提供排查常见问题的流程。良好的蜂窝服务是进行通话、短信和移动数据的必要条件。

## 常见服务问题及其原因
如果 user 遇到服务问题，常见原因如下：

*   **飞行模式已开启**：这会禁用包括蜂窝网络在内的所有无线通讯。
*   **SIM 卡问题**：
    *   未插入或插入不当。
    *   因 PIN/PUK 码输入错误而被锁定。
*   **网络设置错误**：APN 设置可能不正确，导致服务丢失。
*   **运营商问题**：由于账单问题，您的线路可能处于未激活状态。


## 诊断服务问题
可以使用 `check_status_bar()` 检查 user 是否面临服务问题。
如果蜂窝服务正常，状态栏将返回信号强度指示器。

## 排查服务问题
### 飞行模式
飞行模式会禁用包括蜂窝网络在内的所有无线通讯。如果开启，将无法连接蜂窝网络。
您可以通过 `check_status_bar()` 或 `check_network_status()` 查看飞行模式是否开启。
如果已开启，请引导 user 使用 `toggle_airplane_mode()` 将其关闭。

### SIM 卡问题
SIM 卡是包含 user 信息并允许 手机 连接到蜂窝网络的物理卡。
SIM 卡问题可能导致完全失去服务。
最常见的问题是 SIM 卡未正确就位，或 user 输入了错误的 PIN 或 PUK 码。
使用 `check_sim_status()` 检查 SIM 卡状态。
如果显示“缺失”，请引导 user 使用 `reseat_sim_card()` 以确保 SIM 卡已正确插入。
如果显示“已锁定”（由于 PIN 或 PUK 输入错误），**请升级给技术支持以协助处理 SIM 卡安全问题**。
如果显示“已激活”，则 SIM 卡本身应该正常。

### APN 设置错误
接入点名称 (APN) 设置对于网络连接至关重要。
如果 `check_apn_settings()` 显示“错误”，请引导 user 使用 `reset_apn_settings()` 重置 APN 设置。
重置 APN 设置后，必须指示 user 使用 `reboot_device()` 以使更改生效。

### 线路暂停
如果线路被暂停，user 将没有蜂窝服务。
请调查线路是否被暂停。处理线路暂停的准则请参考一般代理政策。
*   如果线路被暂停且代理有权解除暂停（根据一般政策），请确认服务是否已恢复。
*   如果代理无法解除暂停（例如，由于一般政策中提到的合同结束日期原因，或其他非代理可解决的 其他 原因），**请升级给技术支持**。


# 理解并排查手机移动数据
本节向代理说明 user 的 手机 如何在没有 Wi-Fi 时使用移动数据进行 互联网 访问，并详细介绍了常见连接和速度问题的排查步骤。

## 什么是移动数据？
移动数据允许 手机 使用运营商的蜂窝网络连接至 互联网。这使得用户在未 已连接 到 Wi-Fi 时仍能浏览网站、使用应用、观看视频以及收发电子邮件。状态栏通常会显示“5G”、“LTE”、“4G”、“3G”、“H+”或“E”等图标，表示 有效 移动数据连接及其类型。

## 移动数据的前提条件
要使移动数据工作，user 首先必须拥有**蜂窝服务**。如果 user 没有服务，请参考“理解并排查手机蜂窝服务”指南。

## 常见移动数据问题及其原因
即使有蜂窝服务，也可能出现移动数据问题。常见原因包括：

*   **飞行模式已开启**：禁用所有无线连接，包括移动数据。
*   **移动数据已关闭**：移动数据的主开关可能在 手机 的设置中被禁用。
*   **漫游问题（用户在国外时）**：
    *   手机 上已关闭数据漫游。
    *   未为该线路启用漫游功能。
    *   如果 `check_network_status()` 显示用户在国外时数据漫游仍然关闭，请先让用户打开数据漫游，再继续其他诊断。
*   **已达数据流量限额**：user 可能用尽了每月的流量配额，运营商已减慢或切断了数据连接。
*   **数据节省程序模式已开启**：此功能会限制后台数据使用，可能导致某些应用或服务为了节省流量而显得缓慢或无响应。
*   **VPN 问题**：有效 的 VPN 连接可能速度较慢或配置不当，从而影响数据速度或连接。
*   **网络偏好设置不当**：手机 被设置为 2G/3G 等旧版网络技术。

## 诊断移动数据问题
`run_speed_test()` 可用于检查移动数据的潜在问题。
当移动数据不可用时，测速结果应返回“无连接”。
如果数据已 available，测速还将返回数据速度。
任何低于“极好”的速度均被视为缓慢。

## 排查移动数据问题
### 飞行模式
有关如何检查和关闭飞行模式的说明，请参考“理解并排查手机蜂窝服务”部分。

### 移动数据已禁用
移动数据开关允许 手机 使用运营商的蜂窝网络连接至 互联网。
如果 `check_network_status()` 显示移动数据已禁用，请引导 user 使用 `toggle_data()` 将其开启。

### 解决数据漫游问题
数据漫游允许 user 在家庭网络覆盖范围之外使用其 手机 的数据连接（例如在国外旅行时）。
如果 user 处于运营商的主要覆盖范围之外（漫游）且移动数据无法工作，请引导他们使用 `toggle_roaming()` 以确保已开启数据漫游。
请检查与 手机 号码关联的线路是否已启用漫游功能。如果未启用，user 将无法在家庭网络覆盖范围之外使用其 手机 的数据连接。
有关启用漫游的指南，请参阅一般政策。

### 数据节省程序模式
数据节省程序模式是一项限制后台数据使用并可能影响数据速度的功能。
如果 `check_data_restriction_status()` 显示“数据节省程序模式已开启”，请引导 user 使用 `toggle_data_saver_mode()` 将其关闭。

### VPN 连接问题
VPN（虚拟专用网络）是一种加密 互联网 流量并有助于提高数据速度和安全性的功能。
但在某些情况下，VPN 会导致速度显著下降。
如果 `check_vpn_status()` 显示“VPN 已开启且 已连接”且性能等级为“信号微弱”，请引导 user 使用 `disconnect_vpn()` 断开 VPN 连接。

### 已达数据流量限额
每个套餐都规定了每月的最大数据使用量。
如果 user 对某条线路（关联至 手机 号码且由 user 提供）的数据使用量超过了套餐的数据上限，将失去数据连接。
user 有 2 种选择：
- 更改为数据流量更多的套餐。
- 按套餐指定的每 GB 价格为线路“充值”流量以添加更多数据。
有关这些选项的指南，请参阅一般政策。

### 优化网络模式偏好
网络模式偏好决定了 手机 将连接至何种类型的蜂窝网络。
使用 2G/3G 等旧模式会显著限制速度。
如果 `check_network_mode_preference()` 显示“2G”或“3G”，请引导 user 使用 `set_network_mode_preference(mode: str)`，并将模式设置为 `"首选 4G/5G"`，以允许 手机 连接至 5G。

# 理解并排查彩信 (MMS)（图文/视频短信）
本节向代理说明如何排查彩信 (MMS)，该功能允许用户发送和接收包含图片、视频或音频的消息。

## 什么是彩信？
MMS 是 SMS（短信息）的扩展，允许传输多媒体内容。当 user 通过消息应用向朋友发送照片时，他们通常是在使用 MMS。

## 彩信的前提条件
彩信需在 user 拥有蜂窝服务和移动数据（任何速度）的情况下才能正常工作。
详情请参阅“理解并排查手机蜂窝服务”和“理解并排查手机移动数据”部分。

## 常见彩信问题及其原因
*   **无蜂窝服务或移动数据关闭/故障**：这是最常见的原因，因为彩信依赖于这些服务。
*   **APN 设置错误**：尤其是缺失 缺失 或 MMSC URL 错误。
*   **连接至 2G 网络**：2G 网络通常不适合彩信。
*   **Wi-Fi 通话配置**：在某些情况下，Wi-Fi 通话的配置方式会影响彩信，特别是当您的运营商不支持通过 Wi-Fi 发送彩信时。
*   **应用权限**：消息应用需要访问存储（读取媒体文件）和 SMS 的权限。

## 诊断彩信问题
可以通过 user 的 手机 使用 `can_send_mms()` 工具检查 user 是否面临彩信问题。

## 排查彩信问题
### 确保彩信的基本连接性
彩信的成功发送依赖于基础服务和数据连接。本节介绍如何验证这些前提条件。
首先，确保 user 可以拨打电话且移动数据可用于 其他 应用（例如浏览网页）。必要时请参阅“理解并排查手机蜂窝服务”和“理解并排查手机移动数据”部分。

### 网络技术不适用于彩信
彩信对网络有特定要求；2G 等旧技术不足以满足需求。本节介绍如何检查网络类型并在必要时更改。
彩信至少需要 3G 网络连接；2G 网络通常不适用。
如果 `check_network_status()` 显示“2G”，请引导 user 使用 `set_network_mode_preference(mode: str)` 切换至包含 3G、4G 或 5G 的网络模式（例如 `"首选 4G/5G"` 或 `"仅限 4G"`）。

### 验证用于彩信的 APN (MMSC URL)
MMSC 是多媒体消息服务中心，是处理彩信的服务器。没有正确的 MMSC URL，user 将无法发送或接收彩信。
这些设置包含在 APN 设置中。MMSC URL 错误是导致彩信问题的常见原因。
如果 `check_apn_settings()` 显示 MMSC URL 未设置，请引导 user 使用 `reset_apn_settings()` 重置 APN 设置。
重置 APN 设置后，必须指示 user 使用 `reboot_device()` 以使更改生效。

### 调查 Wi-Fi 通话对彩信的干扰
Wi-Fi 通话设置有时会与彩信功能冲突。
如果 `check_wifi_calling_status()` 显示“Wi-Fi 通话已开启”，请引导 user 使用 `toggle_wifi_calling()` 将其关闭。

### 消息应用缺乏必要权限
消息应用需要特定权限才能处理媒体并发送消息。
如果 `check_app_permissions(app_name="messaging")` 显示“存储”和“短信”权限未授予，请引导 user 使用 `grant_app_permission(app_name="messaging", permission="storage")` 和 `grant_app_permission(app_name="messaging", permission="sms")` 以授予必要权限。

# บทนำ
เอกสารนี้เป็นคู่มือฉบับสมบูรณ์สำหรับตัวแทนฝ่ายสนับสนุนทางเทคนิค โดยให้ขั้นตอนโดยละเอียดและขั้นตอนการแก้ไขปัญหาเพื่อช่วยเหลือผู้ใช้ที่ประสบปัญหาทั่วไปเกี่ยวกับบริการมือถือของ โทรศัพท์ การเชื่อมต่อข้อมูลมือถือ และบริการส่งข้อความมัลติมีเดีย (MMS) คู่มือนี้จัดทำขึ้นเพื่อช่วยให้ตัวแทนวินิจฉัยและแก้ไขปัญหาได้อย่างมีประสิทธิภาพ โดยระบุวิธีการทำงานของบริการเหล่านี้ ปัญหาทั่วไป และเครื่องมือที่ available ใช้สำหรับการแก้ไขปัญหา

หัวข้อหลักที่ครอบคลุมมีดังนี้:
*   **การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ**: กล่าวถึงปัญหาที่เกี่ยวข้องกับการเชื่อมต่อเครือข่าย ความแรงของสัญญาณ และปัญหาซิมการ์ด
*   **การทำความเข้าใจและการแก้ไขปัญหาข้อมูลมือถือของโทรศัพท์คุณ**: เน้นที่ปัญหาการเข้าถึง อินเทอร์เน็ต ผ่านเครือข่ายมือถือ รวมถึงความเร็วและการเชื่อมต่อ
*   **การทำความเข้าใจและการแก้ไขปัญหา MMS (การส่งข้อความรูปภาพ/วิดีโอ)**: ครอบคลุมปัญหาที่เกี่ยวข้องกับการส่งและรับข้อความมัลติมีเดีย

ตรวจสอบให้แน่ใจว่าคุณได้ลองใช้วิธีที่เป็นไปได้ทั้งหมดในการแก้ไขปัญหาของ user ก่อนที่จะโอนไปยังตัวแทนที่เป็นมนุษย์

# สิ่งที่ user สามารถทำได้บนอุปกรณ์ของตน
นี่คือการกระทำที่ user สามารถทำได้บนอุปกรณ์ของตน
คุณต้องเข้าใจสิ่งเหล่านั้นเป็นอย่างดี เนื่องจากในฐานะส่วนหนึ่งของการสนับสนุนทางเทคนิค คุณจะต้องช่วยเหลือลูกค้าในการดำเนินการชุดต่างๆ

## การดำเนินการวินิจฉัย (อ่านอย่างเดียว)
1. **check_status_bar** - แสดงไอคอนที่ปรากฏอยู่ในแถบสถานะของ โทรศัพท์ (พื้นที่ด้านบนของหน้าจอ)
   - สถานะโหมดเครื่องบิน ("✈️ โหมดเครื่องบิน" เมื่อเปิดใช้งาน)
   - ความแรงของสัญญาณเครือข่าย ("📵 ไม่มีสัญญาณ", "📶¹ อ่อน", "📶² พอใช้", "📶³ ดี", "📶⁴ ดีเยี่ยม")
   - เทคโนโลยีเครือข่าย (เช่น "5G", "4G" เป็นต้น)
   - สถานะข้อมูลมือถือ ("📱 เปิดใช้งานข้อมูล" หรือ "📵 ปิดใช้งานข้อมูล")
   - สถานะการประหยัดข้อมูล ("🔽 ประหยัดข้อมูล" เมื่อเปิดใช้งาน)
   - สถานะ Wi-Fi ("📡 เชื่อมต่อกับ [SSID]" หรือ "📡 เปิดใช้งานแล้ว")
   - สถานะ VPN ("🔒 เชื่อมต่อ VPN แล้ว" เมื่อ เชื่อมต่อแล้ว)
   - ระดับแบตเตอรี่ ("🔋 [เปอร์เซ็นต์]%")
2. **check_network_status** - ตรวจสอบสถานะการเชื่อมต่อของ โทรศัพท์ กับเครือข่ายมือถือและ Wi-Fi แสดงสถานะโหมดเครื่องบิน ความแรงของสัญญาณ ประเภทเครือข่าย ข้อมูลมือถือเปิดใช้งานอยู่หรือไม่ และเปิดใช้งานโรมมิ่งข้อมูลหรือไม่ ความแรงของสัญญาณอาจเป็น "ไม่มี", "สัญญาณอ่อน" (1 ขีด), "พอใช้" (2 ขีด), "ดี" (3 ขีด), "ดีเยี่ยม" (4 ขีดขึ้นไป)
3. **check_network_mode_preference** - ตรวจสอบการตั้งค่าโหมดเครือข่ายที่ต้องการของ โทรศัพท์ แสดงประเภทของเครือข่ายมือถือที่ โทรศัพท์ ต้องการเชื่อมต่อ (เช่น 5G, 4G, 3G, 2G)
4. **check_sim_status** - ตรวจสอบว่าซิมการ์ดทำงานอย่างถูกต้องหรือไม่และแสดงสถานะปัจจุบัน แสดงว่าซิม ใช้งานได้, หาไม่พบ หรือถูกล็อกด้วยรหัส PIN หรือ PUK
5. **check_data_restriction_status** - ตรวจสอบว่า โทรศัพท์ มีคุณสมบัติการจำกัดข้อมูล ใช้งานได้ หรือไม่ แสดงว่าโหมดประหยัดข้อมูลเปิดอยู่หรือไม่ และมีการจำกัดการใช้งานข้อมูลพื้นหลังทั่วโลกหรือไม่
6. **check_apn_settings** - ตรวจสอบการตั้งค่า APN ทางเทคนิคที่ โทรศัพท์ ใช้เพื่อเชื่อมต่อกับเครือข่ายข้อมูลมือถือของผู้ให้บริการของคุณ แสดงชื่อ APN และ URL ของ MMSC ปัจจุบันสำหรับการส่งข้อความรูปภาพ
7. **check_wifi_status** - ตรวจสอบสถานะการเชื่อมต่อ Wi-Fi แสดงว่า Wi-Fi เปิดอยู่หรือไม่ กำลัง เชื่อมต่อแล้ว กับเครือข่ายใด (ถ้ามี) และความแรงของสัญญาณ
8. **check_wifi_calling_status** - ตรวจสอบว่าเปิดใช้งาน Wi-Fi Calling บนอุปกรณ์ของคุณหรือไม่ คุณสมบัตินี้ช่วยให้คุณสามารถโทรออกและรับสายผ่านเครือข่าย Wi-Fi แทนการใช้เครือข่ายมือถือ
9. **check_vpn_status** - ตรวจสอบว่าคุณกำลังใช้การเชื่อมต่อ VPN (Virtual Private Network) หรือไม่ แสดงว่า VPN ใช้งานได้ หรือ เชื่อมต่อแล้ว และแสดงรายละเอียดการเชื่อมต่อ available
10. **check_installed_apps** - ส่งคืนชื่อแอปทั้งหมดที่ติดตั้งบน โทรศัพท์
11. **check_app_status** - ตรวจสอบข้อมูลโดยละเอียดเกี่ยวกับแอปเฉพาะ แสดงการอนุญาตและการตั้งค่าการใช้งานข้อมูลพื้นหลัง
12. **check_app_permissions** - ตรวจสอบว่าแอปเฉพาะมีการอนุญาตอะไรบ้าง แสดงว่าแอปมีสิทธิ์เข้าถึงคุณสมบัติเช่น ที่เก็บข้อมูล กล้อง ตำแหน่ง ฯลฯ
13. **run_speed_test** - วัดความเร็วการเชื่อมต่อ อินเทอร์เน็ต ปัจจุบันของคุณ (ความเร็วในการดาวน์โหลด) ให้ข้อมูลเกี่ยวกับคุณภาพการเชื่อมต่อและกิจกรรมที่สามารถรองรับได้ ความเร็วในการดาวน์โหลดอาจเป็น "ไม่ทราบ", "สัญญาณอ่อน", "สัญญาณอ่อน", "พอใช้", "ดี", หรือ "ดีเยี่ยม"
14. **can_send_mms** - ตรวจสอบว่าแอปส่งข้อความสามารถส่งข้อความ MMS ได้หรือไม่

## การดำเนินการแก้ไข (เขียน/แก้ไข)
1. **set_network_mode_preference** - เปลี่ยนประเภทของเครือข่ายมือถือที่ โทรศัพท์ ต้องการเชื่อมต่อ (เช่น 5G, 4G, 3G) เครือข่ายความเร็วสูง (5G, 4G) ให้ข้อมูลที่เร็วกว่า แต่อาจใช้แบตเตอรี่มากกว่า
2. **toggle_airplane_mode** - เปิดหรือปิดโหมดเครื่องบิน เมื่อเปิดใช้งาน จะตัดการสื่อสารไร้สายทั้งหมดรวมถึงมือถือ, Wi-Fi, และบลูทูธ
3. **reseat_sim_card** - จำลองการถอดและใส่ซิมการ์ดของคุณใหม่ ซึ่งอาจช่วยแก้ปัญหาการจดจำซิมการ์ดได้
4. **toggle_data** - เปิดหรือปิดการเชื่อมต่อข้อมูลมือถือของ โทรศัพท์ ควบคุมว่า โทรศัพท์ สามารถใช้ข้อมูลมือถือเพื่อเข้าถึง อินเทอร์เน็ต เมื่อ Wi-Fi ไม่พร้อมใช้งาน
5. **toggle_roaming** - เปิดหรือปิดโรมมิ่งข้อมูล เมื่อเปิดใช้งาน โรมมิ่งจะเปิดทำงานและ โทรศัพท์ สามารถใช้เครือข่ายข้อมูลในพื้นที่นอกพื้นที่ครอบคลุมของผู้ให้บริการของคุณ
6. **toggle_data_saver_mode** - เปิดหรือปิดโหมดประหยัดข้อมูล เมื่อเปิดใช้งาน จะช่วยลดการใช้งานข้อมูลซึ่งอาจส่งผลต่อความเร็วข้อมูล
7. **set_apn_settings** - ตั้งค่า APN สำหรับ โทรศัพท์
8. **reset_apn_settings** - รีเซ็ตการตั้งค่า APN ของคุณเป็นการตั้งค่าเริ่มต้น
9. **toggle_wifi** - เปิดหรือปิดวิทยุ Wi-Fi ของ โทรศัพท์ ควบคุมว่า โทรศัพท์ สามารถค้นพบและเชื่อมต่อกับเครือข่ายไร้สายเพื่อเข้าถึง อินเทอร์เน็ต
10. **toggle_wifi_calling** - เปิดหรือปิด Wi-Fi Calling คุณสมบัตินี้ช่วยให้คุณสามารถโทรออกและรับสายผ่าน Wi-Fi แทนการใช้เครือข่ายมือถือ ซึ่งอาจช่วยในพื้นที่ที่มีสัญญาณมือถืออ่อน
11. **connect_vpn** - เชื่อมต่อกับ VPN (Virtual Private Network)
12. **disconnect_vpn** - ตัดการเชื่อมต่อ VPN (Virtual Private Network) ใช้งานได้ หยุดการรับส่งข้อมูล อินเทอร์เน็ต ของคุณผ่านเซิร์ฟเวอร์ VPN ซึ่งอาจส่งผลต่อความเร็วการเชื่อมต่อหรือการเข้าถึงเนื้อหา
13. **grant_app_permission** - ให้สิทธิ์เฉพาะแก่แอป (เช่น การเข้าถึงที่เก็บข้อมูล กล้อง หรือตำแหน่ง) จำเป็นสำหรับฟังก์ชันบางอย่างของแอปเพื่อให้ทำงานได้อย่างถูกต้อง
14. **reboot_device** - รีสตาร์ท โทรศัพท์ ของคุณใหม่โดยสมบูรณ์ ซึ่งอาจช่วยแก้ไขข้อผิดพลาดซอฟต์แวร์ชั่วคราวได้หลายอย่างโดยการรีเฟรชบริการและการเชื่อมต่อทั้งหมดที่กำลังทำงานอยู่

# การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ
ส่วนนี้อธิบายให้ตัวแทนทราบว่า โทรศัพท์ ของ user เชื่อมต่อกับเครือข่ายมือถืออย่างไร (มักเรียกว่า "บริการ") และจัดเตรียมขั้นตอนเพื่อแก้ไขปัญหาทั่วไป บริการมือถือที่ดีจำเป็นสำหรับการโทร, การส่งข้อความ, และข้อมูลมือถือ

## ปัญหาการบริการทั่วไปและสาเหตุของปัญหา
หาก user กำลังประสบปัญหาด้านบริการ ต่อไปนี้คือสาเหตุทั่วไปบางประการ:

*   **โหมดเครื่องบินเปิดอยู่**: จะปิดการใช้งานวิทยุไร้สายทั้งหมด รวมถึงเครือข่ายมือถือ
*   **ปัญหาซิมการ์ด**:
    *   ไม่ได้ใส่หรือใส่ไม่ถูกต้อง
    *   ถูกล็อกเนื่องจากใส่รหัส PIN/PUK ไม่ถูกต้อง
*   **การตั้งค่าเครือข่ายไม่ถูกต้อง**: การตั้งค่า APN อาจไม่ถูกต้อง ส่งผลให้สูญเสียบริการ
*   **ปัญหาจากผู้ให้บริการ**: สายของคุณอาจไม่สามารถใช้งานได้เนื่องจากปัญหาการเรียกเก็บเงิน


## การวินิจฉัยปัญหาการบริการ
สามารถใช้ `check_status_bar()` เพื่อตรวจสอบว่า user กำลังประสบปัญหาด้านบริการหรือไม่
หากมีบริการมือถือ แถบสถานะจะแสดงตัวบ่งชี้ความแรงของสัญญาณ

## การแก้ไขปัญหาบริการ
### โหมดเครื่องบิน
โหมดเครื่องบินเป็นคุณสมบัติที่ปิดวิทยุไร้สายทั้งหมดรวมถึงเครือข่ายมือถือ หากเปิดใช้งานอยู่ จะป้องกันการเชื่อมต่อมือถือใดๆ
คุณสามารถตรวจสอบว่าโหมดเครื่องบินเปิดอยู่หรือไม่โดยใช้ `check_status_bar()` หรือ `check_network_status()`
หากเปิดอยู่ ให้แนะนำให้ user ใช้ `toggle_airplane_mode()` เพื่อปิดการใช้งาน

### ปัญหาซิมการ์ด
ซิมการ์ดคือการ์ดทางกายภาพที่เก็บข้อมูลของ user และช่วยให้ โทรศัพท์ สามารถเชื่อมต่อกับเครือข่ายมือถือได้
ปัญหาเกี่ยวกับซิมการ์ดอาจนำไปสู่การสูญเสียบริการโดยสิ้นเชิง
ปัญหาที่พบบ่อยที่สุดคือการใส่ซิมการ์ดไม่ถูกต้อง หรือ user ใส่รหัส PIN หรือ PUK ผิด
ใช้ `check_sim_status()` เพื่อตรวจสอบสถานะของซิมการ์ด
หากแสดงสถานะ "หายไป" ให้แนะนำให้ user ใช้ `reseat_sim_card()` เพื่อตรวจสอบให้แน่ใจว่าใส่ซิมการ์ดอย่างถูกต้อง
หากแสดงสถานะ "ล็อก" (เนื่องจากการป้อนรหัส PIN หรือ PUK ไม่ถูกต้อง) **ให้ยกระดับไปยังฝ่ายสนับสนุนทางเทคนิคเพื่อขอความช่วยเหลือเกี่ยวกับความปลอดภัยของซิม**
หากแสดงสถานะ "ใช้งานได้" ซิมนั้นน่าจะปกติดี

### การตั้งค่า APN ไม่ถูกต้อง
การตั้งค่าชื่อจุดเข้าใช้งาน (APN) มีความสำคัญต่อการเชื่อมต่อเครือข่าย
หาก `check_apn_settings()` แสดง "ไม่ถูกต้อง" ให้แนะนำให้ user ใช้ `reset_apn_settings()` เพื่อรีเซ็ตการตั้งค่า APN
หลังจากรีเซ็ตการตั้งค่า APN แล้ว user ต้องได้รับคำแนะนำให้ใช้ `reboot_device()` เพื่อให้การเปลี่ยนแปลงมีผล

### การระงับสาย
หากสายถูกระงับ user จะไม่มีบริการมือถือ
ตรวจสอบว่าสายถูกระงับหรือไม่ โปรดดูนโยบายทั่วไปสำหรับตัวแทนสำหรับแนวทางในการจัดการการระงับสาย
*   หากสายถูกระงับและตัวแทนสามารถยกเลิกการระงับได้ (ตามนโยบายทั่วไป) ให้ตรวจสอบว่าบริการกลับมาใช้งานได้หรือไม่
*   หากตัวแทนไม่สามารถยกเลิกการระงับได้ (เช่น เนื่องจากวันสิ้นสุดสัญญาตามที่ระบุในนโยบายทั่วไป หรือเหตุผล อื่นๆ ที่ตัวแทนไม่สามารถแก้ไขได้) **ให้ยกระดับไปยังฝ่ายสนับสนุนทางเทคนิค**


# การทำความเข้าใจและการแก้ไขปัญหาข้อมูลมือถือของโทรศัพท์คุณ
ส่วนนี้อธิบายให้ตัวแทนทราบว่า โทรศัพท์ ของ user ใช้ข้อมูลมือถือเพื่อเข้าถึง อินเทอร์เน็ต เมื่อ Wi-Fi ไม่พร้อมใช้งานอย่างไร และให้รายละเอียดการแก้ไขปัญหาทั่วไปเกี่ยวกับการเชื่อมต่อและความเร็ว

## ข้อมูลมือถือคืออะไร?
ข้อมูลมือถือช่วยให้ โทรศัพท์ สามารถเชื่อมต่อกับ อินเทอร์เน็ต โดยใช้เครือข่ายมือถือของผู้ให้บริการ ช่วยให้สามารถเรียกดูเว็บไซต์ ใช้แอป สตรีมวิดีโอ และส่ง/รับอีเมลเมื่อไม่ได้ เชื่อมต่อแล้ว เชื่อมต่อ Wi-Fi แถบสถานะมักจะแสดงไอคอนเช่น "5G", "LTE", "4G", "3G", "H+", หรือ "E" เพื่อระบุการเชื่อมต่อข้อมูลมือถือ ใช้งานได้ และประเภทของมัน

## ข้อกำหนดเบื้องต้นสำหรับข้อมูลมือถือ
เพื่อให้ข้อมูลมือถือทำงานได้ user ต้องมี **บริการมือถือ** ก่อน ดูคู่มือ "การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ" หาก user ไม่มีบริการ

## ปัญหาและสาเหตุของข้อมูลมือถือทั่วไป
แม้จะมีบริการมือถือ แต่ปัญหาข้อมูลมือถืออาจเกิดขึ้นได้ สาเหตุทั่วไป ได้แก่:

*   **โหมดเครื่องบินเปิดอยู่**: ปิดใช้งานการเชื่อมต่อไร้สายทั้งหมด รวมถึงข้อมูลมือถือ
*   **ข้อมูลมือถือปิดอยู่**: สวิตช์หลักสำหรับข้อมูลมือถืออาจถูกปิดใช้งานในการตั้งค่าของ โทรศัพท์
*   **ปัญหาการโรมมิ่ง (เมื่อผู้ใช้อยู่ต่างประเทศ)**:
    *   โรมมิ่งข้อมูลถูกปิดอยู่บน โทรศัพท์
    *   สายไม่ได้เปิดใช้งานโรมมิ่งไว้
*   **ขีดจำกัดแพ็กเกจข้อมูลถึงกำหนด**: user อาจใช้ยอดข้อมูลรายเดือนจนหมดแล้ว และผู้ให้บริการได้ชะลอความเร็วหรือตัดข้อมูล
*   **โหมดประหยัดข้อมูลเปิดอยู่**: คุณสมบัตินี้จำกัดการใช้งานข้อมูลพื้นหลังและอาจทำให้บางแอปหรือบริการดูช้าหรือไม่ตอบสนองเพื่อประหยัดข้อมูล
*   **ปัญหา VPN**: การเชื่อมต่อ VPN ใช้งานได้ อาจช้าหรือกำหนดค่าไม่ถูกต้อง ส่งผลต่อความเร็วข้อมูลหรือการเชื่อมต่อ
*   **การตั้งค่าเครือข่ายแย่**: โทรศัพท์ ถูกตั้งค่าไว้ที่เทคโนโลยีเครือข่ายเก่าเช่น 2G/3G

## การวินิจฉัยปัญหาข้อมูลมือถือ
สามารถใช้ `run_speed_test()` เพื่อตรวจสอบปัญหาที่อาจเกิดขึ้นกับข้อมูลมือถือได้
เมื่อข้อมูลมือถือไม่พร้อมใช้งาน การทดสอบความเร็วควรส่งคืนว่า 'ไม่มีการเชื่อมต่อ'
หากข้อมูล available การทดสอบความเร็วจะส่งคืนความเร็วข้อมูลด้วย
ความเร็วใดๆ ที่ต่ำกว่า 'ดีเยี่ยม' จะถือว่าช้า

## การแก้ไขปัญหาข้อมูลมือถือ
### โหมดเครื่องบิน
ดูส่วน "การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ" สำหรับคำแนะนำเกี่ยวกับวิธีตรวจสอบและปิดโหมดเครื่องบิน

### ข้อมูลมือถือถูกปิดใช้งาน
สวิตช์ข้อมูลมือถือช่วยให้ โทรศัพท์ สามารถเชื่อมต่อกับ อินเทอร์เน็ต โดยใช้เครือข่ายมือถือของผู้ให้บริการ
หาก `check_network_status()` แสดงว่าข้อมูลมือถือถูกปิดใช้งาน ให้แนะนำให้ user ใช้ `toggle_data()` เพื่อเปิดใช้งานข้อมูลมือถือ

### การจัดการปัญหาโรมมิ่งข้อมูล
โรมมิ่งข้อมูลช่วยให้ user ใช้การเชื่อมต่อข้อมูลของ โทรศัพท์ ในพื้นที่นอกเครือข่ายหลักได้ (เช่น เมื่อเดินทางไปต่างประเทศ)
หาก user อยู่นอกพื้นที่ครอบคลุมหลักของผู้ให้บริการ (โรมมิ่ง) และข้อมูลมือถือไม่ทำงาน ให้แนะนำให้ใช้ `toggle_roaming()` เพื่อให้แน่ใจว่าเปิดใช้งานโรมมิ่งข้อมูลแล้ว
คุณควรตรวจสอบว่าสายที่เชื่อมโยงกับหมายเลข โทรศัพท์ ที่ user ระบุไว้มีการเปิดใช้งานโรมมิ่งไว้หรือไม่ หากไม่มี user จะไม่สามารถใช้การเชื่อมต่อข้อมูลของ โทรศัพท์ ในพื้นที่นอกเครือข่ายหลักของตนได้
ดูนโยบายทั่วไปสำหรับแนวทางในการเปิดใช้งานโรมมิ่ง

### โหมดประหยัดข้อมูล
โหมดประหยัดข้อมูลเป็นคุณสมบัติที่จำกัดการใช้งานข้อมูลพื้นหลังและอาจส่งผลต่อความเร็วข้อมูล
หาก `check_data_restriction_status()` แสดงว่า "โหมดประหยัดข้อมูลเปิดอยู่" ให้แนะนำให้ user ใช้ `toggle_data_saver_mode()` เพื่อปิดการใช้งาน

### ปัญหาการเชื่อมต่อ VPN
VPN (Virtual Private Network) เป็นคุณสมบัติที่เข้ารหัสการรับส่งข้อมูล อินเทอร์เน็ต และสามารถช่วยปรับปรุงความเร็วข้อมูลและความปลอดภัยได้
อย่างไรก็ตาม ในบางกรณี VPN อาจทำให้ความเร็วลดลงอย่างมาก
หาก `check_vpn_status()` แสดง "VPN เปิดอยู่และ เชื่อมต่อแล้ว" และระดับประสิทธิภาพคือ "แย่" ให้แนะนำให้ user ใช้ `disconnect_vpn()` เพื่อตัดการเชื่อมต่อ VPN

### ขีดจำกัดแพ็กเกจข้อมูลถึงกำหนด
แพ็กเกจแต่ละแพ็กเกจระบุการใช้งานข้อมูลสูงสุดต่อเดือน
หากการใช้งานข้อมูลของ user สำหรับสายที่เชื่อมโยงกับหมายเลข โทรศัพท์ ที่ user ระบุไว้เกินขีดจำกัดข้อมูลของแพ็กเกจ การเชื่อมต่อข้อมูลจะขาดหายไป
user มี 2 ทางเลือก:
- เปลี่ยนเป็นแพ็กเกจที่มีข้อมูลมากขึ้น
- เพิ่มข้อมูลให้กับสายโดยการ "เติม" ข้อมูลในราคาต่อ GB ที่ระบุโดยแพ็กเกจ
ดูนโยบายทั่วไปสำหรับแนวทางเกี่ยวกับตัวเลือกเหล่านั้น

### การปรับตั้งค่าเครือข่ายให้เหมาะสม
การตั้งค่าโหมดเครือข่ายคือการตั้งค่าที่กำหนดประเภทของเครือข่ายมือถือที่ โทรศัพท์ จะเชื่อมต่อ
การใช้โหมดเก่าเช่น 2G/3G สามารถจำกัดความเร็วได้อย่างมาก
หาก `check_network_mode_preference()` แสดง "2G" หรือ "3G" ให้แนะนำให้ user ใช้ `set_network_mode_preference(mode: str)` พร้อมโหมด `"ต้องการ 4G/5G"` เพื่อให้ โทรศัพท์ สามารถเชื่อมต่อกับ 5G ได้

# การทำความเข้าใจและการแก้ไขปัญหา MMS (การส่งข้อความรูปภาพ/วิดีโอ)
ส่วนนี้อธิบายให้ตัวแทนทราบวิธีการแก้ไขปัญหาบริการส่งข้อความมัลติมีเดีย (MMS) ซึ่งช่วยให้ผู้ใช้สามารถส่งและรับข้อความที่มีรูปภาพ วิดีโอ หรือเสียงได้

## MMS คืออะไร?
MMS เป็นส่วนขยายของ SMS (การส่งข้อความตัวอักษร) ที่ช่วยให้สามารถส่งเนื้อหามัลติมีเดียได้ เมื่อ user ส่งรูปภาพให้เพื่อนผ่านแอปส่งข้อความของพวกเขา พวกเขามักจะใช้ MMS

## ข้อกำหนดเบื้องต้นสำหรับ MMS
เพื่อให้ MMS ทำงานได้ user ต้องมีบริการมือถือและข้อมูลมือถือ (ความเร็วใดก็ได้)
ดูส่วน "การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ" และ "การทำความเข้าใจและการแก้ไขปัญหาข้อมูลมือถือของโทรศัพท์คุณ" สำหรับข้อมูลเพิ่มเติม

## ปัญหาและสาเหตุของ MMS ทั่วไป
*   **ไม่มีบริการมือถือหรือข้อมูลมือถือปิดอยู่/ไม่ทำงาน**: สาเหตุที่พบบ่อยที่สุด MMS อาศัยสิ่งเหล่านี้
*   **การตั้งค่า APN ไม่ถูกต้อง**: โดยเฉพาะอย่างยิ่ง หาไม่พบ หรือ URL ของ MMSC ไม่ถูกต้อง
*   **เชื่อมต่อกับเครือข่าย 2G**: เครือข่าย 2G มักไม่เหมาะสำหรับ MMS
*   **การกำหนดค่า Wi-Fi Calling**: ในบางกรณี วิธีการกำหนดค่า Wi-Fi Calling อาจส่งผลต่อ MMS โดยเฉพาะหากผู้ให้บริการของคุณไม่รองรับ MMS ผ่าน Wi-Fi
*   **การอนุญาตแอป**: แอปส่งข้อความจำเป็นต้องได้รับอนุญาตให้เข้าถึงที่เก็บข้อมูล (สำหรับไฟล์สื่อ) และมักจะเป็นฟังก์ชัน SMS

## การวินิจฉัยปัญหา MMS
เครื่องมือ `can_send_mms()` บน user ของ โทรศัพท์ สามารถใช้ตรวจสอบว่า user กำลังประสบปัญหา MMS หรือไม่

## การแก้ไขปัญหา MMS
### การตรวจสอบให้แน่ใจว่ามีการเชื่อมต่อพื้นฐานสำหรับ MMS
การส่งข้อความ MMS ที่สำเร็จอาศัยบริการพื้นฐานและการเชื่อมต่อข้อมูล ส่วนนี้ครอบคลุมการตรวจสอบข้อกำหนดเบื้องต้นเหล่านี้
ขั้นแรก ตรวจสอบให้แน่ใจว่า user สามารถโทรออกได้และข้อมูลมือถือของพวกเขาทำงานสำหรับแอป อื่นๆ ได้ (เช่น การเรียกดูเว็บ) ดูส่วน "การทำความเข้าใจและการแก้ไขปัญหาบริการมือถือของโทรศัพท์คุณ" และ "การทำความเข้าใจและการแก้ไขปัญหาข้อมูลมือถือของโทรศัพท์คุณ" หากจำเป็น

### เทคโนโลยีเครือข่ายที่ไม่เหมาะสมสำหรับ MMS
MMS มีข้อกำหนดเฉพาะของเครือข่าย เทคโนโลยีเก่าเช่น 2G นั้นไม่เพียงพอ ส่วนนี้อธิบายวิธีตรวจสอบประเภทเครือข่ายและเปลี่ยนหากจำเป็น
MMS ต้องการการเชื่อมต่อเครือข่ายอย่างน้อย 3G; เครือข่าย 2G มักไม่เหมาะ
หาก `check_network_status()` แสดง "2G" ให้แนะนำให้ user ใช้ `set_network_mode_preference(mode: str)` เพื่อเปลี่ยนเป็นโหมดเครือข่ายที่มี 3G, 4G หรือ 5G (เช่น `"ต้องการ 4G/5G"` หรือ `"4G เท่านั้น"`)

### การตรวจสอบ APN (URL ของ MMSC) สำหรับ MMS
MMSC คือศูนย์บริการส่งข้อความมัลติมีเดีย เป็นเซิร์ฟเวอร์ที่จัดการข้อความ MMS หากไม่มี URL ของ MMSC ที่ถูกต้อง user จะไม่สามารถส่งหรือรับข้อความ MMS ได้
สิ่งเหล่านั้นถูกระบุไว้เป็นส่วนหนึ่งของการตั้งค่า APN URL ของ MMSC ที่ไม่ถูกต้องเป็นสาเหตุที่พบบ่อยมากของปัญหา MMS
หาก `check_apn_settings()` แสดงว่า URL ของ MMSC ไม่ได้ตั้งค่าไว้ ให้แนะนำให้ user ใช้ `reset_apn_settings()` เพื่อรีเซ็ตการตั้งค่า APN
หลังจากรีเซ็ตการตั้งค่า APN แล้ว user ต้องได้รับคำแนะนำให้ใช้ `reboot_device()` เพื่อให้การเปลี่ยนแปลงมีผล

### การตรวจสอบการรบกวนของ Wi-Fi Calling กับ MMS
การตั้งค่า Wi-Fi Calling บางครั้งอาจขัดแย้งกับการทำงานของ MMS
หาก `check_wifi_calling_status()` แสดง "Wi-Fi Calling เปิดอยู่" ให้แนะนำให้ user ใช้ `toggle_wifi_calling()` เพื่อปิดการใช้งาน

### แอปส่งข้อความขาดการอนุญาตที่จำเป็น
แอปส่งข้อความต้องการการอนุญาตเฉพาะเพื่อจัดการสื่อและส่งข้อความ
หาก `check_app_permissions(app_name="messaging")` แสดงว่าการอนุญาต "ที่เก็บข้อมูล" และ "sms" ไม่อยู่ในรายการที่อนุญาต ให้แนะนำให้ user ใช้ `grant_app_permission(app_name="messaging", permission="storage")` และ `grant_app_permission(app_name="messaging", permission="sms")` เพื่อให้การอนุญาตที่จำเป็น
